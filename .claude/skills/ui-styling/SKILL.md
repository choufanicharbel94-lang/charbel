# UI Styling Skill

This is a comprehensive guide for building accessible user interfaces combining shadcn/ui components, Tailwind CSS utility styling, and canvas-based visual design.

## Core Stack Overview

The skill leverages three primary layers:

**Component Layer**: shadcn/ui provides "pre-built accessible components via Radix UI primitives" with a copy-paste distribution model and TypeScript-first approach.

**Styling Layer**: Tailwind CSS offers "utility-first CSS framework" with build-time processing and zero runtime overhead.

**Visual Design Layer**: Canvas-based compositions emphasizing "philosophy-driven design approach" and "minimal text, maximum visual impact."

## Key Use Cases

Deploy this skill when building React-based applications, implementing accessible components like dialogs and forms, creating responsive layouts, establishing design systems, or generating visual materials requiring sophisticated aesthetics.

## Setup Process

Initialize with `npx shadcn@latest init`, add components via `npx shadcn@latest add button card dialog`, then style using Tailwind utilities directly in JSX. Alternative Tailwind-only setup available for Vite projects.

## Documentation Structure

The skill references multiple guides covering component library usage, theming customization, accessibility patterns, Tailwind utilities, responsive design, and visual design systems. Automation scripts assist with component installation and configuration generation.

## Best Practices Summary

Emphasize component composition, utility-first styling, mobile-first responsiveness, accessibility-first development, consistent design tokens, dark mode support, performance optimization, TypeScript integration, intentional visual hierarchy, and expert craftsmanship in UI implementation.
